# Cas Clients SLASHR — Reference

> Ce fichier est la bibliotheque de cas clients utilisee par l'agent pour l'onglet **Cas Clients** de la proposition HTML.

---

## Instructions pour l'agent

L'agent choisit **2-4 cas pertinents** pour le prospect en fonction de :
- **Secteur similaire** (priorite 1)
- **Problematique similaire** (priorite 2) : gap SEO, refonte, multi-marches, dependance paid, etc.
- **Taille comparable** (priorite 3)

Pour chaque cas selectionne, l'agent ecrit :
1. **Situation initiale** — en chiffres, ancree dans la problematique du cas
2. **Ce qu'on a fait** — 1-2 phrases, levier active
3. **Resultats** — chiffres + timeline (toujours conservateurs, toujours sources)
4. **Citation client** — si disponible dans le cas

### Regles

- **Jamais inventer un cas.** Utiliser uniquement ceux de ce fichier.
- **Adapter la presentation** au prospect : mettre en avant l'angle qui resonne avec SA situation.
- **Si aucun cas ne matche parfaitement**, utiliser le plus proche et le cadrer avec : "Resultats observes sur un profil comparable".
- **L'onglet Cas Clients existe toujours** — au minimum 2 cas.

## Ajouter un cas

Pour ajouter un nouveau cas, copier la structure ci-dessus et remplir tous les champs. Les tags permettent a l'agent de matcher rapidement.

**Format des tags :** minuscules, tirets, pas d'espaces. Exemples : `e-commerce`, `b2b`, `migration`, `local`, `dependance-paid`.
